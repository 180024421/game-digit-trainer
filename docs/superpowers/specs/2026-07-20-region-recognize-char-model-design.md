# 区域识别（仍用单字模型）— 设计规格

日期：2026-07-20  
状态：已确认（用户：训练按字切；识别框一块区域出整串）

## 1. 目标

- **训练**：继续单字切图 + 单字 CNN（不变）。
- **识别**：用户框一块区域（蓝 ROI，或一个很宽的绿框）→ 区域内自动切字 → 用已有单字模型逐字识别 → 输出整串。

不做 CRNN / 行级重训；不改 Studio 运行时。

## 2. 识别框解析优先级

点「预览识别」或「用所选模型识别」时：

1. **≥2 个绿字框**且看起来像逐字框 → 直接按绿框顺序识别（用户已手调）。
2. **有蓝 ROI** → 在蓝框内自动切字 → 识别。
3. **仅 1 个很宽的绿框**（宽高比 ≥ 1.8 且宽 ≥ 40px）→ 视作区域，框内自动切字 → 识别。
4. **仅 1 个窄绿框** → 单字识别（兼容原行为）。
5. **都没有** → 尝试整图自动切字；仍无则提示先框蓝框。

切字使用现有 `segment_binary` + `auto_fix_boxes`，间距取界面「间距」旋钮。

## 3. UI

- 识别后把自动切出的绿框画回画布，方便对照。
- 按钮提示改为：蓝框圈数字行即可识别整串；训练仍用逐字框。

## 4. 非目标

- 不引入行模型 / CTC。
- 不保证粘连字一次切对（可调间距、或改为手调多绿框后再识别）。
